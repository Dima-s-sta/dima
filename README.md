## Описание

Графическое приложение "Дневник погоды" для ведения записей о погодных условиях. Позволяет добавлять новые записи с указанием даты (ДД.ММ.ГГГГ), температуры (°C), текстового описания и наличия осадков.
Все данные сохраняются в локальный файл `data.json`, что обеспечивает их сохранность между запусками программы.
Встроенные инструменты фильтрации позволяют быстро находить записи по дате или температуре.
Приложение написано на языке Python с использованием библиотеки Tkinter.


## Требования

Для работы приложения требуется установленный интерпретатор **Python 3.x**. Все используемые библиотеки являются стандартными (Tkinter входит в комплект установки Python).

## Инструкция по запуску

1.  Убедитесь, что на вашем компьютере установлен Python версии 3.6 или новее.
2.  Скачайте или клонируйте репозиторий с проектом.
3.  Откройте терминал или командную строку и перейдите в папку проекта.
4.  Запустите скрипт следующей командой:
   ```bash
   python weather_diary.py
   ```

## Использование приложения

### Добавление новой записи:
1.  Заполните поля в блоке **"Добавить новую запись"**.
2.  Нажмите кнопку **"Добавить запись"**. Запись появится в таблице и будет сохранена в файл.
3.  Поля ввода будут автоматически очищены для следующей записи.

### Фильтрация записей:
Используйте блок **"Фильтр записей"** для поиска нужных данных.
*   **Дата:** Введите конкретную дату (ДД.ММ.ГГГГ), чтобы увидеть все записи за этот день.
*   **Темп. выше:** Введите число (например: `16`), чтобы отобразить только те дни, когда температура была выше +16°C.
*   Нажмите **"Применить фильтр"**.
*   Нажмите **"Сбросить фильтр"**, чтобы вернуть полный список записей.

## Примеры использования и тесты (Сценарии)

Вы можете использовать эти сценарии для проверки корректности работы программы и обработки ошибок.

### Сценарий 1: Успешное добавление записи

*   **Цель:** Проверить базовый функционал добавления данных.
*   **Действия:**
    1.  В поле "Дата" введите: `25.03.2026`.
    2.  В поле "Температура" введите: `+12`.
    3.  В поле "Описание" введите: `Ясно`.
    4.  Выберите радиокнопку "Осадки": `Нет`.
*   **Ожидаемый результат:** После нажатия "Добавить запись" новая строка появится в таблице.

### Сценарий 2: Проверка валидации ввода (Обработка ошибок)

*   **Цель:** Убедиться, что программа не позволяет добавить некорректные данные.
*   **Действия:**
    1.  В поле "Дата" введите: `25-03-26` (неправильный формат).
*   **Ожидаемый результат:** Появится окно ошибки с текстом: *"Дата должна быть в формате ДД.ММ.ГГГГ..."*.
2.  В поле "Температура" введите: `тепло` (не число).
*   **Ожидаемый результат:** Появится окно ошибки с текстом: *"Температура должна быть числом."*.
3.  Оставьте поле "Описание" пустым и нажмите "Добавить запись".
*   **Ожидаемый результат:** Появится окно ошибки с текстом: *"Описание погоды не может быть пустым."*.

### Сценарий 3: Использование фильтрации по дате

*   **Цель:** Найти все записи за конкретный день.
*   **Действия:**
    1.  Убедитесь, что в таблице есть несколько записей за разные даты (например за `28.04.2026` и `29.04.2026`).
    2.  В поле фильтра "Дата" введите: `29.04.2026`.
